What's New - LoLQ

0.9.1

May 24, 2018

- Added support for Flex Queue
- Fixed a bug where a JavaScript error would occur if a game starts but LoLQ hasn't been through a queue yet (thanks WastedNinjaz)

0.9.0

May 22, 2018

OPEN BETA
- API calls now routed via LoLQ server; register for access token at https://www.lolq.org/get-access-token/
- Added "Reset window positions" button to the settings window
- Added a check to see if DWM composition (Aero Glass) is enabled in Windows
- If not -> themes that use transparency will be disabled. A theme must specify whether it uses transparency or not in its theme_common.css (see Default and LoLQv1 themes for examples)
- Added "Queue history" feature
- Added "When match begins" settings option
- Added "Hide to tray until champ select" settings option
- Reworked the UI CSS so it's easier to write new themes
- Also Default theme no longer uses transparency
- Reworked app minimize / focus / clicking on tray icon behaviors
- "Waiting for League" splash screen will now only show if you click on tray icon when League client isn't running
- Added small delay to summoner name detection to prevent misdetects
- Added notification when new version is available for download
- Added "Remove Summoner" button incase there's a misdetection / typo and there's also a summoner by that name
- App will now remember desktop positions of My team / Enemy team windows
- Tray notifications look more stylish now
- Fixed a bug where accounts that have been transfered between servers would show incorrect summoner stats
- Fixed a bug where last10 stats would not get fetched for the other summoners if one summoner had no match history
- Fixed a bug where the app windows would stay hidden if League window was minimized when LoLQ starts
- Fixed a bug where debug files would not get generated if League window wasn't focused
- (internal) More improvements to summoner name detection logic (won't get stuck now if one returns an empty string for whatever reason)
- (internal) Fixed a bug where a javascript error would sometimes occur when exiting the app and a tray notification was open
- (internal) Champ pick autodetect now checks champions in order of MAIN ROLE->PLAY RATE (highest to lowest), based on the latest champion.GG data
- (internal) Fixed a bug where a JavaScript error would trigger after a queue dodge
- (internal) Fixed a bug with clipboard checking of "x joined the lobby"

0.8.9

April 4, 2018

- Fixed a bug in About -window where Kne's name wasn't first in the names being thanked (thanks Kne for finding this bug)
- Major rewrite to champ pick autodetection
- Switched from OCR text reading to just comparing champ icon images
- Faster, more accurate (100%) and consumes less CPU power
- Improvements to summoner name clipboard copy/manual input logic and made sure there's always a backup option working if autodetection fails
- Added "copy debug log + screenshot" button to the app in champ select screen

0.8.5

March 29, 2018

- Changed it so the app windows move freely on the deskop (instead of snapping to client)
- Further improved summoner name autodetection (still needs work)
- Added file logging and improved the overall logging output
- Logfile can be found at %APPDATA%\LoLQ-userdata\lolq-log.txt
- Logfile will be overridden on each app startup
- Starting LoLQ.exe from the windows command line will now produce verbose, colored logs to the console
- Added some mouseover tooltips to the summoner info box
- (internal) Major rewrite to screen reading & OCR logic. Should take less CPU power and work more consistently.
- (internal) Major rewrite to how the summoner data is requested from Riot API (to protect against rate limits while still trying to keep the app responsive).
